📋 Projet Prescription - SIH CHU Andrainjato

Module de prescription médicale – Frontend Next.js & Backend NestJS

Version 1.0 – Documentation générée le 10/05/2026

🔧 Stack technique

Frontend
Next.js 16 + TypeScript + CSS custom (variables CSS)
Backend
NestJS + Prisma 5 + PostgreSQL 18
Authentification
JWT (access token)
Environnement
Kali Linux (terminal uniquement, pas d'IDE graphique)

⚙️ Flux de données

React Form → lib/api.ts → NestJS Controller → Prisma Service → PostgreSQL

Le frontend utilise les fonctions de lib/api.ts pour appeler l'API REST (port 3001). Chaque fonction envoie un objet JSON avec patientId, les champs spécifiques, et le token JWT dans le header.

🔍 Règles métier importantes

  • Les prescriptions sont toujours liées à un patient enregistré (pas d'anonyme).
  • Degré d'urgence : Normal, Urgent, STAT. Les STAT sont traités en priorité.
  • La prescription médicamenteuse peut générer une ordonnance ajustable (quantités par médicament).
  • Les prescriptions en cours sont mises à jour dynamiquement après validation.
  • Les noms de champs côté backend sont stricts (ex: renseignements, pas renseign ; typeEEG, pas eegType ; typeKine...).
